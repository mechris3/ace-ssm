# ACE-SSM: The Glass Box Inference Engine

### "Because in 2026, a conclusion without a rationale is just a guess."

**ACE-SSM** (Architectural Component Ensemble - Situation Specific Model) is a high-performance, transparent reasoning engine designed to move beyond "Black Box" AI. Built for the M3 Max era, it externalizes expert-level cognitive processes into a traceable, reactive, and visually animated state-space search.

---

## 🚀 The Core Vision
Unlike standard LLMs that rely on probabilistic next-token prediction, ACE-SSM utilizes a **Discrete Cognitive Cycle**. It builds an explicit **Situation Specific Model (SSM)** by comparing a general **Task Structure** (the rules) against a domain-specific **Knowledge Base** (the library). 

The result is a "Glass Box" system where every node in the graph is accompanied by a **Rationale Packet**, detailing exactly which heuristics (Urgency, Parsimony, or Cost) drove the engine's "Searchlight" at any given millisecond.

## 🛠️ Tech Stack & Architecture
* **Framework:** Angular (v19+) — Utilizing `OnPush` change detection for high-frequency updates.
* **State Management:** NgRx — Ensuring strict immutability and a perfect "History of Thought" audit trail.
* **Reactive Orchestration:** RxJS — A time-based "Heartbeat" stream governing the inference cycle.
* **Visualization:** D3.js — Real-time, force-directed graph rendering of the evolving mental model.

## 🧠 The Triple-Operator Cycle
The engine operates on a recurring functional pulse:
1.  **Abductive Goal Generation:** Identifies logical gaps between the rules and the current state.
2.  **Strategic Search:** Uses weighted heuristics to prioritize the most critical or parsimonious path.
3.  **Resolution:** Pattern-matches goals against the Knowledge Base or triggers an **Inquiry** to the user.

## 📈 Key Features
* **The Searchlight Effect:** Real-time visual pulse of the engine's current focus area.
* **Strategy Sliders:** Hot-swappable heuristic weights to change the engine's "personality" (e.g., Cautious vs. Aggressive).
* **Human-in-the-Loop:** Seamless "Inquiry" states that pause autopilot to gather missing human data.
* **M3 Max Optimized:** Engineered for concurrent physics simulations and deep reactive state updates.

### 🚦 Getting Started
1.  **Define the Grammar:** Edit `task-structure.json` to set your ontological rules.
2.  **Load the Library:** Populate `knowledge-base.json` with domain facts.
3.  **Initiate Heartbeat:** Toggle the "Run" mode to watch the engine build a theory from scratch.
